These are the classes that actually compute EFPs. If you plan on studying single graphs, EFP would typically be used. For studying EFPs in large collections, use EFPSet for greater efficiency and simplicity.

EFP

A class for a storing and computing a single EFP.

EFP(edges, measure=hadr, beta=1, normed=True, check_type=True, ve_alg=numpy, np_optimize=greedy)

Arguments

  • edges: list, Edges of the EFP graph specified by tuple-pairs of vertices.
  • measure: string, optional, See Measures for options.
  • beta: float, optional, A value greater than zero.
  • normed: bool, optional, Controls energy normalization.
  • check_type: bool, optional, Whether to check the type of the input or use the first input type.
  • ve_alg: string, optional, Controls which variable elimination algorithm is used, either numpy or ef. Leave as numpy unless you know what you're doing.
  • np_optimize: string, optional, When ve_alg='numpy' this is the optimize keyword of numpy.einsum_path

EFPSet

A class for holding collections of EFPs.

EFPSet(*args, filename=None, measure='hadr', beta=1, normed=True, check_type=False, verbose=False)

Arguments

  • filename: string, Edges of the EFP graph specified by tuple-pairs of vertices.
  • measure: string, optional, See Measures for options.
  • beta: float, optional, A value greater than zero.
  • normed: bool, optional, Controls energy normalization.
  • check_type: bool, optional, Whether to check the type of the input or use the first input type.
  • ve_alg: string, optional, Controls which variable elimination algorithm is used, either numpy or ef. Leave as numpy unless you know what you're doing.
  • np_optimize: string, optional, When ve_alg='numpy' this is the optimize keyword of numpy.einsum_path
